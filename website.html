Portfolio Website Project

Welcome! While completing my UX Immersion course at CareerFoundry, I was given the option to specialize in 1 of 3 areas. I choose Frontend Development for designers. By learning and understanding the back-end development and the basics of coding, helps me become a better designer who can relate, understand and better connect with my team members on what goes on after “handoff”. This also allows me to understand the limitations and the complexity of a design, which will help me design more user friendly and realistic interfaces. Therefore I took the challenge to build my portfolio website from scratch, within 5 weeks.

Goals

The goal within this process was to create a portfolio website to:

  • demonstrate my new skills for front end development
  • provide visibility to my UX Immersion project and any other additional ones
  • understand the basics of coding to better connect with my team members
  • allow future employers to learn about me, my work, view my resume and how to contact me

My Design Process

01 - Discover

Problem

A need to provide visibility to allow future employers and my professional network to learn about me, showcase my work, and view my resume from online.

Solution

Build a responsive, well organized, and fully functional website that can be accessed online.

Research:

Researched layouts and designs of portfolio websites on platforms such as Behance, Dribble, Journo Portfolio, and etc..

02 - Define

Personas

Creating personas helped me consider what type of content to include on my portfolio site and what their user journey navigation throughout it might entail.

Who are they?

Recruiter

She is using LinkedIn to scout for someone who meets the company’s requirements. She stumbles across a profile that contains a portfolio website and views it from her desktop. She’s mostly interested in seeing the skills, their resume, and skimming through a summary of their work.

Hiring Manager

He frequently checks his emails from his mobile device and reviews a portfolio website provided to him by a recruiter between meetings. This candidate peeked his interest so he flags the email so that he can review the portfolio thoroughly. He wants to learn about this candidate to make sure they are the right fit and is interested in reading about the person and viewing an applicable projects in detail before contacting them.



03 - Ideate

Wireframes

I created mid-fidelity wireframes of how I wanted the layout of the design to look. I started to build my HTML files following the layout of the wireframes.

Responsiveness

Created wireframes for mobile, tablet, and desktop versions. Starting with a mobile first design, I adjusted the layout of images and text to fit larger viewports.

Mobile Layout
Single column view of all content.
Tablet Layout
Content will move to a two-column layout where applicable and image sizing will adjust on viewports 640px and larger.
Desktop Layout
Content and images will adjust to fit a viewport of 1024px and larger.

Research:

Researched layouts and designs of portfolio websites on platforms such as Behance, Dribble, Journo Portfolio, and etc..

04 - Design & Develop

Design

UI Design

Before creating my CSS style sheet, I defined the typography and chose the colors for the website which helped me stay consistent with the look and feel. My goal was to create a visual identity that’s professional and friendly.

Typography

Dosis - a clean and modern display font will be used for headings and the body.

Buttons

Will have rounded corners with a radius of 45px.

Color Palette

I applied, black, shades of grey, shades of blue and white to my website. The black and grey combination will give a simple, modern, and sophisticated look/style to my website. The shades of blue will give the website a pop of color. I will be using white for my background to help accent the dominant colors. I feel that by doing this, the visitors will be able to focus on my projects without feeling overwhelmed by any additional colors I chose for my app designs. This color palette will also give me freedom to choose my images and avoid them potentially clashing with one another and/or the overall color scheme.

Development

Learning HTML, CSS, and JavaScript

I have previous experience using HTML and CSS, however this is the first time I have used JavaScript and built a functional website from scratch. A challenge for me was trying to incorporate some animation to my images, but after much research, I was able to make the design I wanted work.

To implement CSS styles correctly, I went through my HTML and added class names and IDs to elements to make it easier for me to make changes for the different viewports. I also created various button and hyperlink states in CSS (visited, hover, focus, active) so that users could see the buttons/links change as they progress through the different stats.

Also, I was able to animate a SVG with CSS keyframes. By adding a class to different elements in the SVG, I was able to use CSS to transform the element's color or make it move across the screen. It was a great introduction to CSS animation.

Responsive Design

After using the Mobile first approach, I implemented media queries to my CSS page so that my code could be responsive to the different viewports.

Visual Studio Code and GitHub

I used Visual Studio Code to create my code pushed the code to GitHub for version control. I was able to commit my work and push and pull changes to the main branch, but most importantly, I could rollback and revert changes. Being able to view change history is very helpful not only to myself, but to others that could be working on portions of the code.

05 - Test

Usability Testing

Once the website was functional, it was time for testing and I:

  • created a test plan and script
  • recruited 5 testers
  • presented them with the scenario and tasks to complete
  • compiled errors
  • used Nielson’s 0-4 scale to rate the severity

User Testing Report

Summary

I received positive feedback, the functionality of my website worked. During the usability testing, there were a few discoveries that I rated with minor usability problems and a few cosmetic problems. All were addressed with the exception of it being too wordy, but I did however add a few bullet points to break the paragraph up on the Index and first section of the PlutoPay summary. Some appreciated all the info that was provided. Here is some of the feedback summarized:

    Navigation

    All the participants found the website's navigation clear and easy to use.

    Layout

    All liked the graphics that were used. It drew them in and made my website more enticing.

    Content

    Most participants found the content on the website easy to understand and well-organized, however, some mentioned that it contained too much information and suggested adding bullet points.

    Responsiveness

    The website was very scalable/responsive on a variety of different screen sizes.

    Call to Actions

    Call to actions on the website were easy to find

Evaluations
Rating 2 = Minor usability problem
(2 or more people mentioned these issues)
  • The hamburger menu label was too far on my project page, so they missed it
  • The ‘Top’ button that takes the user back to the top of the page blended in
  • The font was hard to see
Rating 1 = Cosmetic problem
    (1 or more people mentioned these issues)
  • Website was too wordy, suggested adding bullet points
  • The navigation visited link was to dark of a contract against the black header
  • Add more white space in my ‘Hero’ section, the pipe symbol looks like an ‘I’ and blends in
Code Quality & Accessibility
